# 곡선 편집·관리 UI 완성 (Object/Dataset 대화상자)

## 날짜
2026-07-22

## 배경

세미랜드마크 계획([[20260722_237_semilandmark_support_plan]]) 2단계 시점의 곡선
트레이싱 UI는 **추가(그리기)만** 가능했다. 237의 "남은 것"으로 명시된
"곡선 삭제/편집 UI"와 "데이터셋 대화상자 곡선 설정 노출"을 이 날 완성했다.
곡선을 그린 뒤 이름·개수(N)·설명을 붙이고, 점을 옮기고, 삭제하고, 저장 전까지
메모리에 붙들어 두는 전체 편집·관리 흐름이다.

## 작업 내용

### 곡선 정의를 오브젝트 대화상자로 일원화
- 곡선 테이블을 `ObjectDialog`에 추가: 곡선별 이름 / N / 추적 여부(Traced)를
  보여주고 N을 그 자리에서 편집(데이터셋 전역 스킴에 반영). 뒤따르는 곡선의 시작
  인덱스가 자동 재계산.
- 곡선 정의를 데이터셋 대화상자가 아니라 **오브젝트 대화상자에서** 하도록 이동 —
  실제 트레이싱과 같은 화면에서 스킴을 다룬다.
- **트레이스로 스킴을 키운다**: 스킴에 없는 새 곡선을 그리면 막지 않고 스킴을
  확장(그 자리에서 N을 물어 추가).

### 곡선 편집·선택·삭제
- 추적된 곡선의 **원시 점을 편집**: 점을 잡아 이동, 세그먼트 중간을 클릭해 점
  삽입. 곡선 번호 표시.
- 곡선 **선택**: 곡선 근처를 클릭하면 선택(모드 무관), hover 하이라이트.
- **삭제**: 우클릭 컨텍스트 메뉴(곡선 근처에서만), 빈 공간 우클릭은 팬(pan).
  Delete Curve는 오브젝트의 트레이스를 지우되 데이터셋 전역 슬롯은 유지;
  데이터셋 전역 삭제(`delete_curve_from_dataset`)는 슬롯을 지우고 인덱스를
  재번호·리매핑.
- 곡선 모드에서 우드래그 팬 지원.

### 저장 시점 정합성
- 곡선 편집을 **저장 전까지 메모리에 보관**(`curve_raw_map`), 즉시 DB에 쓰지 않음.
  N 변경은 라이브로 반영, 취소 시 미저장 편집을 감지해 확인. (초기엔 즉시 영속화
  했다가, in-memory 모델로 정정.)
- 곡선 이름/설명을 config에 저장, 테이블에서 편집.

### 데이터셋 대화상자 정리
- 데이터셋 대화상자에 **랜드마크 이름·곡선 테이블** 노출, 곡선 개수와 곡선별
  세미랜드마크 수(N)를 분리해 편집.
- 대화상자를 **탭으로 재구성**.

### 곡선 툴버튼/테이블 다듬기
- 곡선 툴버튼을 플랫 스타일로, 라벨을 "Cv"→"Curve"로. 곡선 테이블을
  Name/N/Traced로 슬림화, 컬럼 폭 맞춤, 좌표 입력을 두 테이블 사이로 이동.
- 임의 오브젝트 편집 취소 시 확인 다이얼로그, 곡선 버튼 hover 색.
