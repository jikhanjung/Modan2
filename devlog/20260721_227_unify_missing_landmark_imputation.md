# 결측 랜드마크 대입 통합 및 정확도 수정 (R03 항목 2)

## 날짜
2026-07-21

## 배경

R03 항목 2: 다이얼로그의 "show estimate"는 회전 정합까지 하도록 고쳤지만
(devlog 221), 분석 파이프라인의
`MdDatasetOps.procrustes_superimposition_with_imputation`은 여전히 평균형
좌표를 그대로 복사하고 있었다. 화면에 보이는 추정과 분석에 들어가는 값이
다를 수 있는 상태.

조사해 보니 단순한 불일치가 아니라 **분석 경로 쪽이 심각하게 틀려 있었다**.

## 진단

합성 데이터로 정답을 알고 측정했다: 동일한 모양을 무작위 회전/축척/이동시킨
표본 8개(노이즈 0)를 만들고, 한 객체에서 랜드마크 하나를 제거한 뒤, 완전
데이터 GPA 결과와 비교. 노이즈가 0이므로 이상적인 추정 오차는 0이어야 한다.

**측정 오차: 중심체크기의 61%.**

원인 두 가지:

1. **정렬 전에 대입했다.** 대입이 정렬 루프 *안*, 첫 회전 *이전*에
   일어났다. 그 시점의 객체는 중심 이동·크기 정규화만 되어 있고 회전은
   임의 상태다. 평균형 좌표를 그대로 복사하면 객체의 시작 방향만큼 어긋난
   곳에 찍힌다 — 다이얼로그에서 고쳤던 것과 정확히 같은 종류의 버그.

2. **한 번 대입하면 다시는 갱신되지 않았다.** `estimate_missing_landmarks`는
   `None`인 좌표만 채우는데, 첫 대입 이후로는 `None`이 아니므로 이후 99번의
   호출이 전부 무의미했다. 계측 결과 **호출 48회 중 실제 대입은 1회**.
   docstring이 주장하던 "iteratively refine estimates"는 실제로 일어난 적이
   없었고, 잘못된 값이 관측 데이터인 양 정렬에 참여해 결과를 오염시켰다.
   (`missing_positions` 변수가 계산만 되고 쓰이지 않은 채 남아 있던 것이
   원래 의도의 흔적으로 보인다.)

## 수정

### 1. 공용 헬퍼 `MdModel.impute_missing_landmarks`

두 경로가 같은 수학을 쓰도록 모듈 수준 함수로 추출. 참조 형상을 객체의
**관측된** 랜드마크 위에 similarity 변환(Kabsch 회전 + 축척 + 이동)으로
정합한 뒤 빈 자리를 가져온다. 차원은 참조 형상에서 추론(2D/3D 모두 지원 —
다이얼로그 쪽 구현은 2D 하드코딩이라 3D 데이터셋에서 좌표가 잘렸다).

다이얼로그(`estimate_missing_for_object`)와 분석
경로(`estimate_missing_landmarks`) 모두 이 함수를 호출한다.

### 2. EM 방식 정제 루프

`procrustes_superimposition_with_imputation`을 재구성:

1. 빈 자리를 `None`인 채로 두고 정렬 (평균형은 nanmean, 회전은 각 객체가
   실제로 가진 랜드마크로만) — 지어낸 데이터에 정렬하지 않는다.
2. 수렴 후, 관측 랜드마크에 평균형을 정합해 빈 자리를 채운다.
3. 이제 모든 객체가 완전하므로 재정규화·재정렬한다. 빈 자리가 있던 객체는
   관측 부분집합으로만 중심/크기가 정해져 평균형을 왜곡하고 있었다.
4. 원래 결측이던 위치를 다시 열고 개선된 평균형으로 재추정. 추정값이 움직이지
   않을 때까지 반복(최대 `MAX_IMPUTATION_REFINEMENTS = 5`).

재추정할 때마다 원래 결측 위치를 `None`으로 되돌리므로, 정합은 항상 진짜
관측 랜드마크 위에서만 이루어지고 이전 추정값을 재료로 쓰지 않는다.

정렬 루프 자체는 `_align_to_mean_shape()`로 추출(완전 데이터 경로와 정제
라운드가 공유).

## 결과 (노이즈 0 합성 데이터, 중심체크기 대비 오차)

| 단계 | 오차 |
|---|---|
| 기존 구현 | 61% |
| 정렬 후 대입 (좌표 직접 복사) | 13% |
| + 관측 랜드마크에 평균형 정합 | 1.9% |
| + 정제 라운드 (최종) | **0.0%** |

시드 6개 × 랜드마크 6개 = 36 케이스: 노이즈 0에서 평균/최대 모두 0.000%,
노이즈 0.02에서 평균 1.25% / 최대 3.35% (노이즈 수준 자체와 같은 자릿수 —
형상 변이가 있는 이상 그 이하로는 알 수 없다).

성능: 50객체 × 30랜드마크에서 완전 데이터 15ms, 결측 40개 115ms. 결측이 있는
데이터셋에서만 정제 라운드가 돌고, 분석은 진행 표시가 있는 작업이라 무방.

## 테스트

`tests/test_imputation_accuracy.py` (신규 20개):
- 헬퍼: 0/37/90/-120/180도 회전+축척+이동 하에서 정확 복원, reflection은
  일치시키지 않음(Kabsch에서 반사 제외), 3D, 차원 추론, 공유 랜드마크 부족,
  참조에도 없는 랜드마크, 빈 입력.
- GPA: 랜드마크 6개 각각을 제거해도 오차 0.5% 미만, 결측이 남지 않음,
  완전 데이터셋은 대입 경로를 강제로 태워도 결과 동일, 모든 객체에서 결측인
  랜드마크는 그대로 둠(별도 게이트가 보고).

기존 테스트 1개(`test_estimate_missing_landmarks`)는 좌표 직접 복사를
검증하고 있어 정합 후 값으로 갱신했다: 참조가 (0,0)-(10,10)이고 객체가
(1,1)-(3,3)이면 중간 랜드마크는 (5,5)가 아니라 객체의 중간인 (2,2)에 와야
한다. 옛 기대값은 객체 바깥이었다.

## 남은 것

분석 결과 자체가 달라지는 변경이므로 실제 데이터셋으로 확인 권장. 결측
랜드마크가 없는 데이터셋은 경로가 아예 분리되어 있어 영향 없음(테스트로 고정).
